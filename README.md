# BasuniAI

Оркестратор конституционных ботов для Discord. Каждый бот — агент с собственной таблицей в БД, промптом и инструментами; всё проходит через языковую модель (OpenAI).

## Установка

1. Клонируйте репозиторий и перейдите в каталог проекта.

2. Создайте виртуальное окружение и установите зависимости:

```bash
python -m venv basuni
# Windows (PowerShell):
.\basuni\Scripts\Activate.ps1
# Linux/macOS:
# source basuni/bin/activate

pip install -r requirements.txt
```

3. Настройте конфигурацию:

- Скопируйте `.env.example` в `.env` и заполните:
  - `DISCORD_TOKEN_ELDER` — токен бота «Старейшина» (из [Discord Developer Portal](https://discord.com/developers/applications))
  - `OPENAI_API_KEY` — ключ OpenAI для ответов агента
  - `OPENAI_MODEL` — модель (по умолчанию `gpt-3.5-turbo`; если в проекте есть доступ к `gpt-4o-mini`, задайте `OPENAI_MODEL=gpt-4o-mini`)
  - При необходимости `DATABASE_URL` (по умолчанию — SQLite в `data/basuni.db`)

- В **Discord Developer Portal** у приложения бота включите:
  - **Privileged Gateway Intents**: Message Content Intent (и при необходимости Server Members Intent для проверки ролей).

- Заполните `config/default.yaml`:
  - `guild_id` — ID вашего Discord-сервера (ПКМ по серверу → «Копировать ID», при включённом режиме разработчика).
  - В блоке `channels` укажите ID каналов:
    - `elder_inbox` — канал, куда граждане пишут заявки старейшинам.
    - `elder_decisions` — канал для публикации решений (можно тот же или отдельный).
    - При желании `court_inbox`, `council_inbox` — для уведомлений суда/совета.
  - В блоке `role_ids` укажите ID ролей:
    - `pmj` — роль ПМЖ (граждане с этой ролью могут обращаться к старейшинам). Если оставить `0`, проверка ПМЖ отключена (все могут писать).

## Запуск

Из корня проекта:

```bash
python main.py
```

Запустится бот «Старейшина» (и другие роли из `enabled_roles` в конфиге). БД инициализируется при первом запуске (таблица `elder_cases` и др.).

## Тестирование бота «Старейшина»

1. Убедитесь, что бот добавлен на сервер и имеет право читать/писать в канале `elder_inbox`.
2. Если проверка ПМЖ включена (`role_ids.pmj` не 0), выдайте себе или тестовому пользователю роль ПМЖ.
3. Напишите в канал `elder_inbox` любое обращение, например:
   - «Хочу подать апелляцию по процедуре по делу №1»
   - «Запрос на референдум: вопрос такой-то»
4. Бот должен:
   - создать дело в БД (таблица `elder_cases`) и ответить в тот же канал (через LLM и инструменты).
5. Проверка без ПМЖ: уберите роль ПМЖ у пользователя и напишите в `elder_inbox` — бот должен ответить отказом.

## Структура проекта

- `config/default.yaml` — каналы, роли, привязка по боту.
- `prompts/elder_system.md` — системный промпт старейшины.
- `src/core/` — конфиг, БД, агент, инструменты, модели (в т.ч. `ElderCase`).
- `src/roles/elder/` — бот старейшины, его инструменты и логика.
- `docs/` — архитектура и статус разработки.

## База данных

- По умолчанию: SQLite, файл `data/basuni.db`.
- У каждого бота своя локальная таблица. У старейшин — `elder_cases` (обращения, решения, запрет повторного вмешательства).
- Таблицы создаются при первом запуске оркестратора.
