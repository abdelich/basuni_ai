# Чек-лист: что сделать для тестирования старейшины

Всё необходимое для теста уже реализовано: ПМЖ, БД, каналы/роли в контексте агента, инструменты. Ниже — что заполнить и в каком порядке делать.

---

## 1. Discord: приложение и бот

1. Зайди в [Discord Developer Portal](https://discord.com/developers/applications) → **New Application** (или выбери существующую).
2. Вкладка **Bot** → **Add Bot**.
3. В разделе **Privileged Gateway Intents** включи:
   - **Message Content Intent** (обязательно — иначе бот не видит текст сообщений);
   - **Server Members Intent** (чтобы в инструментах были роли и участники).
4. **Reset Token** → скопируй токен (один раз показывается). Это будет `DISCORD_TOKEN_ELDER`.

---

## 2. Файл `.env`

1. В корне проекта скопируй `.env.example` в `.env`.
2. В `.env` заполни:

```env
DISCORD_TOKEN_ELDER=сюда_вставь_токен_бота
OPENAI_API_KEY=sk-...твой_ключ_openai
```

- Без `OPENAI_API_KEY` бот ответит, что «не настроен API ключ» (но подключится к Discord).
- `DATABASE_URL` можно не трогать — по умолчанию будет `data/basuni.db`.

---

## 3. Добавить бота на сервер

1. В Developer Portal: твоё приложение → **OAuth2** → **URL Generator**.
2. Scopes: отметь **bot**.
3. Bot Permissions: отметь минимум **Read Message History**, **Send Messages**, **View Channels**, **Read Messages/View Channels**. При проверке ПМЖ по ролям нужен **Manage Roles** или хотя бы просмотр участников (даёт Server Members Intent).
4. Скопируй сгенерированный URL, открой в браузере, выбери сервер и добавь бота.

---

## 4. Включить режим разработчика в Discord

1. Discord (клиент) → **Настройки** → **Расширенные** → **Режим разработчика** — **Вкл**.
2. Тогда можно копировать ID: ПКМ по серверу/каналу/роли → **Копировать ID**.

---

## 5. Заполнить `config/default.yaml`

Нужны только те поля, которые реально используешь. Остальные можно оставить `0`.

| Поле | Где взять | Пример |
|------|-----------|--------|
| **guild_id** | ПКМ по названию сервера → Копировать ID | `123456789012345678` |
| **channels.elder_inbox** | ПКМ по каналу, куда будут писать заявки → Копировать ID | `111111111111111111` |
| **channels.elder_decisions** | ПКМ по каналу для решений (можно тот же, что inbox) → Копировать ID | `111111111111111111` или другой |
| **role_ids.pmj** | ПКМ по роли «ПМЖ» (или как она у тебя называется) → Копировать ID. Если оставить **0** — проверка ПМЖ выключена, писать смогут все | `222222222222222222` или `0` |

Остальные каналы (`court_inbox`, `council_inbox` и т.д.) и роли можно оставить `0`, пока не нужны.

---

## 6. Запуск

Из корня проекта (желательно с активированным venv):

```bash
# Windows PowerShell
.\basuni\Scripts\Activate.ps1
python main.py
```

Должно появиться что-то вроде: «Запуск бота: elder», «Старейшина: inbox канал …». БД создастся при первом запуске (`data/basuni.db`, таблица `elder_cases`).

---

## 7. Как тестировать

1. Зайди на сервер в Discord и открой канал, ID которого указан в **elder_inbox**.
2. Напиши любое сообщение, например:  
   **«Хочу подать апелляцию по процедуре»** или **«Запрос на референдум: нужно ли принять правило X?»**
3. Ожидаемое поведение:
   - Если **pmj = 0**: бот создаёт дело в БД и отвечает в тот же канал (через LLM и инструменты; при наличии OpenAI ответ будет осмысленным).
   - Если **pmj** задан и у тебя есть эта роль: то же самое.
   - Если **pmj** задан и у тебя нет роли: бот ответит, что обращаться могут только граждане с ПМЖ.
4. Дополнительно: напиши что-то вроде «опубликуй решение по делу №1: подтвердить процесс» — модель может вызвать инструмент `publish_decision`; тогда в канале решений (elder_decisions) появится сообщение и дело закроется в БД.

---

## 8. Если что-то не работает

| Симптом | Что проверить |
|--------|----------------|
| Бот не подключается | Токен в `.env`, бот добавлен на сервер, интенты включены в Developer Portal. |
| Бот онлайн, но не реагирует на сообщения | `guild_id` и `elder_inbox` в `default.yaml` совпадают с твоим сервером и каналом; Message Content Intent включён. |
| «Только граждане с ПМЖ» при наличии роли | В `role_ids.pmj` указан ID именно той роли, что выдана пользователю; у бота есть доступ к списку участников (Server Members Intent). |
| Ответ «не настроен API ключ» | В `.env` задан `OPENAI_API_KEY`. |
| Ошибка при создании БД | Папка `data/` создаётся автоматически; проверь права на запись в каталог проекта. |

---

## Реализовано для теста

- Оркестратор, конфиг (.env + YAML), БД (SQLite, таблица `elder_cases`).
- Проверка ПМЖ (вкл/выкл через `role_ids.pmj`).
- Создание дела при каждом обращении в inbox, передача номера дела агенту.
- Агент с инструментами: get_channels, get_roles_and_members, send_message_to_channel, publish_decision, get_case, list_elder_cases.
- В контекст агенту передаются каналы и роли сервера; ответ идёт в тот же канал, решения — в канал решений (или куда выберет модель через send_message_to_channel).
- Запрет повторного решения по одному делу (elder_already_decided в БД).

Этого достаточно, чтобы полноценно протестировать старейшину на своём сервере.
